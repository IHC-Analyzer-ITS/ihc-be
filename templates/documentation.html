API Documentation

1. Submit Data

POST /data

Submits data to the server.

Request:

Content-Type: application/json

Example:

{
    "name": "John Doe",
    "total_data": 10,
    "sync_status": "Synced",
    "description": "Sample description"
}

Response:

{
    "message": "Data received successfully",
    "name": "John Doe",
    "timestamp_create": "2024-09-23 15:30:00",
    "total_data": 10,
    "sync_status": "Synced",
    "description": "Sample description"
}

Status code: 200 (OK) or 400 (Bad Request)

2. Get Data

GET /data

Fetches the existing data stored in the server.

Response:

[
    {
        "name": "John Doe",
        "timestamp_create": "2024-09-23 15:30:00",
        "total_data": 10,
        "sync_status": "Synced",
        "description": "Sample description"
    }
]

Status code: 200 (OK) or 400 (Bad Request)

3. Capture Image

POST /capture/<name>

Captures an image using the server's camera for the specified user.

Request:

Parameter in URL: name (the user name)

Example: /capture/johndoe

Response:

{
    "success": "Captured image successfully",
    "image_path": "projects/johndoe/image_1.jpg"
}

Status code: 200 (OK) or 400/500 (Error)

4. Get Project Images

GET /projects/<name>

Fetches all images for a specific user project.

Request:

Parameter in URL: name (the user name)

Example: /projects/johndoe

Response:

[
    {
        "title": "image_1.jpg",
        "url": "http://localhost:5000/images/johndoe/image_1.jpg"
    },
    {
        "title": "image_2.jpg",
        "url": "http://localhost:5000/images/johndoe/image_2.jpg"
    }
]

Status code: 200 (OK) or 400 (Bad Request)
